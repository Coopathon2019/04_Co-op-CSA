Co-op CSA 的 NSDB
項目細部描述解決方案的主要特色與描述
主要要滿足的需求或解決的問題為何？ * 台灣友善小農耕種面積小，產量不穩定，多樣性不足，時常被既有的機通路排除在外，因此，要自己行銷、販售、配送農產品，疲於奔命； * 友善農夫勞力、財力與技術都有限，缺少互助機制、適當的機器以及充足的知識來管理自己的田地。 * 社區居民雖然想要支持在地小農，卻苦無門路。 這個解決方案包括三個行動： # 建立合作農場作為小農的技術、知識、勞力以及農產品的集散地（hub），我們會製作合作農場操作手冊，說明如何運用合作社的原則，迅速建立社區合作農場作為社員與社員、社員與生產者連結的基地； # 建構 Co-op CSA 系統，協助管理社員資料、社員貢獻、小農報單以及社員訂購、小農知識交換、機械分享使用登記…等等； # 進入社區，舉辦合作農場說明會
主要的受益者即使用後的效益是什麼？ * 對於農夫（從事農務的人員）來說：可以持續提高收入、減輕勞務 * 對於支持者來說：可以獲得最新鮮的食物，完全透明的生產過程，依據使用程度獲得分潤 * 對於加盟（社間合作）的小農來說：可以享有優惠的通路費，也可以做 PGS，技術交流… * 對農場來說：共同擁有、共同購買、共同規劃、共同承擔風險、團結力量大 * 對於社區來說：環境教育、社區廚房、food education、社區認同凝聚，形成更大的生態農業支持力量
這個解決方案的獨特點 * 以合作社的運作方式，在「共同的需求」上，尋求合作 * 透過一個個的 Co-op CSA 連結既有的小農， Co-op CSA 是合作社，初期，在不投入人事費的狀況下就可以開始進行小農連結與服務的工作 * 降低社區農場成立的門檻（無論在金錢、技術與產量上面，都可以隨遇而安，不用強求） * Co-op CSA 的支持力量可以達到偏遠的農村地區，無論在購買農產品，或是參訪、連結…
相關的產品、服務、活動以及需要的技術、知識…等等 * 需要說服消費者支持購買「蔬菜箱」的理由以及優點 * 需要一個可以讓合作小農填寫預估產量的系統 * 需要一個可以彙整所有小農以及 Co-op CSA 農場一週產量的訂單、出貨記錄、付費的系統 * 需要一個記錄生產者、Co-op CSS 農場人員、消費者的貢獻的遊戲規則與記錄系統 * 需要深入了解合作社的價值與原則 * 需要了解 CSA
需要那些配套才能成功導入？ * 需要一本[[如何建立合作農場]]手冊以及搭配的系統（Co-op CSA 配菜系統） * 需要先建立一個示範合作農場（目前已經在新竹幻象 CSA 基地試運作快一年了） * 需要有完整的合作社價值、原則的教育課程（主婦聯盟已經有完整的資料）
